Upgrade Forge — Prefab Authoring Guide

How to edit UpgradeForgeModuleCell.prefab in the Unity editor so the mod's runtime wiring picks it up. Companion to the interaction code in Features/Forge/UpgradeForgeBehavior.cs.

The runtime contract

The metem pipeline cannot carry game script components, so the prefab communicates with the mod purely through named transforms and plain Unity components (colliders, meshes, lights). At runtime, UpgradeForgeBehavior.BuildInteractables() scans the built module for these names (exact, case-sensitive):

NameRequiredWhat the runtime does with it
RelicTubeTargetyes (any count; 6 currently)Relic mount point. Gets a click target; inserted relics dock here (base pivot aligned to the anchor's position and rotation).
InputTargetyes (one)Module socket. A carried BuildBox is loaded here; empty-handed clicks commit the upgrade (when no CommitTarget exists).
CommitTargetoptional (one)Dedicated commit button. Empty-handed click = commit. Strongly recommended — without it, committing means clicking the module-socket rim.

Each of the anchors above also supports three optional child objects:

Child nameComponent(s)Behavior
ClickTargetany Collider (Is Trigger ✓)Used as the click region instead of the auto-generated default box. A collider placed directly on the anchor itself works too.
Highlightmesh / light, disabledEnabled while the player's crosshair is on the interactable, disabled otherwise. Substitute for the vanilla socket outline.
Filledmesh / light, disabledEnabled while a relic (or the BuildBox, on InputTarget) is docked on that anchor.
Note Layers are forced by the mod at runtime (InteractiveObjects), because the editor project's layer table doesn't match the game's. Leave everything on Default — don't try to set layers in the editor.

1 · Open the prefab

  1. Open the VoidCrewUnityEditor/VoidCrewUnityEditor project in Unity Hub.
  2. In the Project window, navigate to Assets/Mods/VoidCrewTerminus/.
  3. Double-click UpgradeForgeModuleCell to enter Prefab Mode (the isolated editing view). All edits below happen here; Unity saves them back to the prefab asset when you exit (or with Auto Save on).

2 · Add the CommitTarget anchor

  1. In the Hierarchy, right-click the prefab root (UpgradeForgeModuleCell) → Create Empty.
  2. Rename it exactly CommitTarget.
  3. With the move tool (W), place it on whatever part of the mesh should read as the commit button/lever — somewhere visually distinct from the tubes and the module socket.
  4. Orientation doesn't matter for a button (nothing docks on it), position does.
Tip Parent new anchors directly under the prefab root, not under FBX child nodes. The FBX nodes carry tiny non-uniform import scales (0.01 / 0.033…) that make hand-editing sizes and offsets confusing. The existing RelicTubeTarget/InputTarget anchors live under FBX nodes — that works (the runtime compensates for scale on generated colliders), but unit-scale parents are much easier to author against.

3 · Author precise click colliders

Without these, the mod generates plain boxes (0.35 m on tubes, 1.2 m on the socket, 0.3 m on the commit button). To match click regions to the visible geometry:

  1. Right-click an anchor (e.g. a RelicTubeTarget) → Create Empty → rename it ClickTarget.
  2. Add Component → Box Collider (or Capsule Collider for round tubes).
  3. Check Is Trigger. This matters twice: players shouldn't collide with click regions, and solid colliders would occlude other interactables.
  4. Click Edit Collider and drag the handles until the green wireframe hugs the tube/button mesh. The gizmo shows the true world-space result, so inherited FBX scale is a non-issue here — what you see is what the raycast hits.
  5. Repeat per anchor. A collider placed directly on the anchor GameObject (instead of a ClickTarget child) is also honored.
Gotcha The player's interaction ray is blocked by solid (non-trigger) colliders. If the module's collision mesh sits in front of a click target, clicks hit the hull instead. Keep click targets slightly proud of the collision surface, or recess the collision mesh around them.

4 · Hover + filled indicators

Highlight (hover feedback)

  1. Under an anchor, create a child named exactly Highlight.
  2. Give it something visible: an emissive ring/quad mesh around the tube mouth, a slightly scaled duplicate of the tube mesh with an emissive HDRP material, or a small Point Light.
  3. Disable the GameObject (uncheck the box next to its name in the Inspector). The mod enables it while hovered and disables it again on look-away.

Filled (docked-state feedback)

  1. Same recipe, child named exactly Filled, disabled by default.
  2. Shown while a relic is docked in that tube (or while the BuildBox occupies InputTarget). A lit tube cap or status lamp reads well from across the room.
  3. An anchor can have both — Highlight tracks the crosshair, Filled tracks contents.

5 · Anchor placement rules

  • Rotation is meaningful on dock anchors. Docked items are aligned base-pivot-to-anchor, matching the anchor's rotation — the anchor's +Y is the docked relic's "up". Tilt the anchor, and the relic mounts tilted.
  • Leave clearance at InputTarget. The BuildBox is the largest docked item; its mesh shouldn't clip the forge body, and players need to see and grab it back out.
  • Tube count is free. All RelicTubeTargets are registered; Phase 3 uses at most 4 (hardcoded capacity), Phase 5 raises capacity with Forge level — six tubes is forward-compatible.
  • Names are load-bearing. RelicTubeTarget, InputTarget, CommitTarget, ClickTarget, Highlight, Filled — exact and case-sensitive. A typo silently gets ignored (check the BepInEx log line described below).
  • Plain Unity components only. Game scripts (sockets, interactables, etc.) can't ride the metem bundle; the only game-package component on the prefab should remain the VoidCrewAsset marker on the root.

6 · Export and deploy

  1. Exit Prefab Mode and save.
  2. In the Project window, select the folder Assets/Mods/VoidCrewTerminus (the folder itself, exactly one selection).
  3. Menu bar → Void Crew → Export Asset Bundle.
  4. The bundle lands in Exported Assets/ at the Unity project root as voidcrewterminus.metem + voidcrewterminus.metem.manifest.
  5. Copy both files to the repo's Assets/ directory (tracked in git), and next to VoidCrewTerminus.dll in the game's BepInEx/plugins/… folder — AssetLoader loads every *.metem sitting beside the DLL.
  6. No C# rebuild is needed for prefab-only changes.

7 · Verify in game

  • On building the Forge, the BepInEx log should print [Forge] Built interactables: N relic tubes, module socket, commit button. — if the tube count or commit button is missing, an anchor name didn't match.
  • Hover a tube: HUD prompt appears and your Highlight child lights up.
  • Insert a relic (click a tube while carrying one): relic mounts on the anchor, Filled lights, chat shows “Relic inserted (n/4)…”.
  • Grab a docked relic: it comes off, Filled goes dark, !forgestatus agrees.
  • Carry a BuildBox to InputTarget, click: box docks. Empty-handed click on CommitTarget: commit fires with a chat result.
