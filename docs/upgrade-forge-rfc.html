Upgrade Forge — RFC

RFC VoidCrewTerminus mod Design only

Design-level proposal for the Upgrade Forge — a new ship module that extends progression past level 3, up to level 10, and adds three roguelike subsystems (tiered relics, gambling perks, escalating sectors). This document is for designers, players, and stakeholders who know the game's systems but not the mod's implementation. For the technical companion, see upgrade-forge-design.html.

Summary

This RFC proposes the Upgrade Forge: a new installable ship module that lets a crew push their weapons, shields, reactors, and other modules past the current level-3 ceiling, all the way to level 10. Upgrading is done by physically inserting relics into the Forge. Most relics are now tiered (Common / Rare / Legendary), and some are cursed — they pay better but make the module they touch operationally fragile in interesting ways. The system layers on top of vanilla Void Crew without changing what already works.

Motivation

Today's late run hits a ceiling fast. Once your modules are L3 and your hold is full of relics, there's not much to invest in. Relics become "items I pick up to craft a thing later" rather than the centerpiece of progression. The TODO file at the project root has the seed of this idea — "new leveling system for weapons (or ship modules in general); lean into the roguelike aspects of the game; levels 4-10 use relics to upgrade."

This RFC turns that note into a coherent system. The aim is to:

  • Give the late run something meaningful to fight for, without inflating numbers past recognition.
  • Make relics feel like loot worth hunting — not just crafting fodder.
  • Add roguelike texture: gambling moments, risk/reward, run-defining choices.
  • Keep the runs themselves clean — no meta-progression, no save files, no permanent unlocks.

Goals & Non-Goals

Goals

  • Extend module progression to L10 in a way that feels earned — not granted.
  • Make individual relic drops matter: tier, identity, and "do I curse my favorite turret with this?" decisions.
  • Introduce risk/reward via cursed relics — the cost is operational, not statistical.
  • Preserve the existing roguelike loop: everything resets when the ship dies.
  • Don't break vanilla. L1–3 module behavior is untouched.

Non-Goals

  • No meta-progression. No persistent module levels across runs. No save format change.
  • No new in-game resources. Only existing currencies are used: relics and alloys.
  • No replacement of the existing BuildBox / module-swap upgrade flow. Vanilla L1→L2→L3 still happens the way it does today.
  • No new enemies, biomes, or POIs in this proposal. (The TODO file lists those separately; they are out of scope here.)

Module Levels (3 → 10)

Today every module caps at L3. This proposal adds levels 4 through 10, accessible only through the Upgrade Forge. The vanilla L1→L2→L3 progression continues to work exactly as it does now — when your crew finishes that, they bring the L3 module to the Forge to push further.

Each level above 3 is a small, deterministic stat bump (e.g. damage, fire rate, output, shield capacity) plus — possibly — a perk roll. Stats are tuned so that an L10 module is meaningfully stronger than an L3, but not so strong that L4–L6 feel like wasted intermediate steps.

The Forge

The Upgrade Forge is a new ship module the crew installs like any other. Physically, it has two interaction surfaces:

  • Relic slots — crew carry relics to it and insert them. The Forge holds up to its Forge Capacity at once (which depends on the Forge's own level).
  • Alloy terminal — crew spend alloys here to push the Forge Meter (see below). This mirrors the Fabricator's existing self-upgrade pattern, so it should feel familiar.

Upgrading a module: walk relics to the Forge, target a module, commit. The relics are consumed. The module gains a level — and if the relics' tiers match an empty perk slot on the module, a perk is rolled.

Diegetic, not menu-driven Insertion is in-world — you physically carry a relic to the Forge, you physically put it in. There's no inventory menu for the relic side. The alloy terminal is a screen because alloys are abstract; relics aren't.

Relic Tiers

The game today has ~25 distinct relics, and they're functionally interchangeable as crafting inputs. This RFC adds a tier property to each:

Common Rare Legendary

Tier is a quality dial, not a gate. Any tier can be used for any upgrade level. The difference is what the relic delivers:

  • Stat magnitude — a Legendary relic produces a bigger stat bump than a Common at the same upgrade level.
  • Perk-roll chance — higher tier = better odds of rolling a perk.
  • Perk slot eligibility — the higher-rarity perk slots can only be filled by higher-tier relics (see below).

Drop rates are biased by Sector Escalation: early sectors are mostly Common; deeper sectors mix in Rare and Legendary.

Perks & Slots

Each module has 3 perk slots, each gated by tier:

SlotRequired tierDescription
Slot 1Any (Common+)The first perk a module ever rolls.
Slot 2Rare or LegendaryA Common upgrade after Slot 1 is full does nothing perk-wise — just the flat stat bump.
Slot 3Legendary onlyA fully-perked module requires at least one Legendary relic somewhere along its path.

Perks are committed first, revealed after. You don't see what you're getting. You insert the relics, you commit, you live with the result. This is intentional — it preserves the "did I just find a treasure?" tension that picking from a menu would dissolve.

Perks are authored content. Each module category (weapons, reactors, shields, engines, scanners) has its own perk pool. A handful of flagship relics (~5–10 of the 25) carry signature perks that only roll when that specific relic is the one inserted. The rest of the relics draw from the shared category pool.

Forge Meter

The Forge has its own level. The Forge's level determines its Capacity — how many relics it can hold at once, which in turn caps the maximum module-upgrade level you can attempt (since L10 needs 4 relics inserted simultaneously, you need a Forge that holds 4).

The Forge gains levels through the Forge Meter, which fills from two sources:

  • Sector progression (passive) — each successful jump bumps the meter by a fixed amount.
  • Alloy investment (active) — spend alloys at the Forge terminal to add to the meter.

Each level threshold is larger than the last (roughly 1.5× growth). The numbers are tuned so that passive progression alone isn't enough to fully level the Forge — to unlock L10 module upgrades, the crew must invest alloys. This makes the Forge a real strategic choice: spend alloys on hull, on the Fabricator, on the Forge?

Why it matters Without this dial, the Forge would just be "use it whenever you have relics." With it, the Forge is a commitment: every alloy you spend here is an alloy you didn't spend on something else. That tension is the whole point.

Sector Escalation

Each successful sector jump scales the run in two ways:

  • Enemy threat — HP, damage, and (optionally) density rise smoothly. Deeper jumps are tougher.
  • Loot quality — the relic tier distribution shifts. Sector 1 drops are almost all Common; by sector 5+, Rare/Legendary become common.

Together these create a classic roguelike push: deeper means more dangerous AND more rewarding. The decision "do we jump again or extract?" gets real weight.

Cursed Relics

Roughly 30% of relics are cursed. A cursed relic still works — the stat bump is full strength, the perk roll still happens — but if a perk lands, it carries a Maintenance Burden.

Burden is operational, not statistical A burdened module is not weaker on paper. It just becomes annoying to operate. It might cut out for a few seconds at random. It might require a manual reset every so often. It might radiate heat to neighbors. The cost shows up as crew work, not as a debuff in a tooltip.

Example perks (illustrative — final names/values during content authoring):

PerkBenefitBurden
Overcharged+30% damagePowers off for 2–4 seconds every 30–90 seconds
Vampiric5% damage healed to ship hullModule radiates heat; nearby modules tick warmer
Erratic+50% fire rate~5% chance per shot to require a manual reset

The intent is to create emergent gameplay. A crew with three Overcharged turrets is choosing chaos — when the swarm shows up, someone is running ladder-to-ladder reactivating things.

A Sample Run

To make this concrete, here's how a run might unfold with the system in place.

Sector 1 — Departure

You install the Upgrade Forge in the back bay before launch. It starts at L1, Capacity 1. Your starboard turret is already L3 from the BuildBox kit. You haven't found any relics yet.

Sector 2 — First relic

Looting a derelict, you find a Common relic. You carry it back to the Forge, slot it in, target the starboard turret, and commit. Turret goes L3→L4. Damage tick up. No perk rolls this time. The Meter advances 20 from the jump.

Sector 3 — The Forge levels

Another two jumps and a few alloys spent at the Forge terminal push the Meter past 100. Forge L2, Capacity 2. You can now do L6 upgrades. You pick up two Common relics and push the starboard turret to L6. Slot 1 fills with "Hyperflux" — a small reload-speed bonus. Things are getting good.

Sector 5 — The fork in the road

You find a Legendary. It glows red — cursed. You hesitate. A cursed Legendary is exactly what you'd need to fill Slot 3 on something — but the burden will follow that module for the rest of the run. You commit it to the starboard turret. Roll: "Overcharged." +30% damage. The turret will now blink off for a few seconds at random.

Next combat encounter, the turret cuts out mid-fight. Someone runs back to reset it. The team learns the timing. The fight feels different.

Sector 7 — The L10

By sector 7 the Forge is L4, Capacity 4. You've been hoarding for this. Four relics — three Rare, one Legendary — go into the Forge, target the starboard turret. Commit. L9→L10. Slot 2 was already full; Slot 3 gets a "Stormpiercer" signature roll because the Legendary you used was a flagship relic. The turret is now genuinely terrifying. You also have a small ongoing maintenance load from the earlier curse. Tradeoffs everywhere.

Sector 9 — Death

A bad jump, a swarm you can't burn down. Ship dies. All of it — the L10 turret, the Forge level, the relic stash, the perks — wipes. Next run starts fresh. That's the deal.

16 relics total to fully push one module from L3 to L10. Suggested tiers are advisory — you can use any tier, you just won't get the higher perk slots and won't get the bigger stat bumps. A typical run yields fewer than 16 relics in total, which means crews must choose which module to commit to. That's the design.

Open Questions

These are decisions deferred to playtesting rather than locked at design time.

  • Cursed ratio. 30% of relics cursed is the default. Could be 20% (curses are special) or 50% (curses are the norm).
  • Forge visual identity. What does the Forge look like? Footprint, animation, particle effects. Should it visibly "heat up" as the Meter fills?
  • Burden taxonomy. "Random shutoffs" is the seed. Other event types — heat ticks, reset interactions, power oscillations — to be authored alongside the cursed perks.
  • Player communication. How does the player know a module is L7 vs L10? Visible glyphs? Glowing intensity? UI tooltip on look?
  • Capacity gate UX. When the Forge is L2 and the player tries an L10 upgrade, what does the rejection look like? A clear message? A locked icon on the higher-cost levels?
  • Multiplayer feel. When a crewmate commits an upgrade, do all players see the same reveal animation? Does the host's roll get a small UI flourish on every screen?

Risks & Mitigations

RiskMitigation
L10 modules trivialize the endgame; encounters become one-sided. Cost ramp is steep enough (16 relics) that one or two modules per good run reach L10. Sector Escalation also pushes enemy difficulty alongside, so the curve evens out. Tune at playtest.
Cursed relics feel mandatory — the bonus is so good players always take the burden. Make burdens genuinely costly in active gameplay (forces crew rotation away from other duties). If still mandatory after playtest, reduce the stat bonus or make the burden more disruptive.
Cursed relics feel like trash — the burden is so bad nobody uses them. If playtest shows curse-avoidance, sweeten the bonus (e.g. cursed Legendaries always roll a perk; clean ones might not).
Players hoard relics for the perfect upgrade and never commit during a run. The Forge has limited capacity early. Holding relics doesn't buy you anything if you can't insert them yet. Combined with per-run reset, hoarding is actively bad.
New players can't figure out the system. In-world Forge has clear interaction prompts. Tooltips on relic tiers (color coding). Possibly a tutorial relic at run start that demonstrates the flow.
Multiplayer feels desynced — clients see different module states. Host is the only authority on rolls; clients receive results. (Implementation detail covered in the technical doc.)

Alternatives Considered

Extend the existing Fabricator tier system to L10

Void Crew's Fabricator already has a real numeric tier and an L3 cap. We could just raise that cap. Rejected because it would upgrade the crafting machine, not the modules it produces. Weapons would still be L1/L2/L3 prefab swaps; nothing about the actual ship modules would change.

Author 7 new prefabs per weapon (L4–L10 variants)

The "right" way to extend the existing BuildBox swap mechanic. Rejected for content scope — that's 7 × (number of weapons + reactors + shields + ...) new assets, and balancing them is its own multi-month project.

Meta-progression (modules persist across runs)

An L10 turret you keep forever. Rejected because it dilutes the roguelike loop. Void Crew's identity is "everything dies when the ship dies." A persistent overlay undermines that.

Treat all relics as fungible currency

Simpler economy: any relic upgrades any module by some amount, no tiers, no signatures. Rejected because it kills the "this specific relic just dropped, what do I do with it?" moment that's the whole point of relics as a loot category.

Visible perk preview before committing (Slay-the-Spire style)

Show three possible perks, let the player pick one. Rejected because it removes the gamble. We want "what did I just get?" not "what should I shop for?" — they're very different game feels.

A custom upgrade currency separate from relics

Introduce a new pickup ("upgrade cores" or similar) for L4–L10. Rejected because it crowds the loot table and adds a second loot economy without enriching the existing one. Relics doing double duty is more elegant.
